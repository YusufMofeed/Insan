# Overview

This document defines the UI/UX specification of the Insan platform.

The design focuses on storytelling, emotional clarity, and minimal distraction.

The interface is built around one core concept:

> A Journey is not a profile. It is a human experience.

---

# 1. Design Philosophy

## 1.1 Human-Centered Design

The UI must always prioritize the human story over interface elements.

---

## 1.2 Minimalism

- No unnecessary visual noise
- No heavy animations
- No distractions from content

---

## 1.3 Emotional Neutrality

The design should be respectful and calm:
- No aggressive colors
- No gamification elements
- No sensational visuals

---

## 1.4 Content First

Content is always the primary focus.

UI elements are secondary support layers.

---

This is not a profile page.

It is a **Journey Page**.

---

- Full Name
- "In Brief" summary
- Basic identity information

---

### 5.2 Biography Section

Narrative text describing the person.

---

### 5.3 Life Journey Section

---

“What they dreamed of”

---

### 5.5 Skills Section

“What they were capable of”

---

### 5.6 Achievements Section

“What they accomplished”

---

### 5.7 Stories Section

---

- Family

---

### 5.9 Memories Section

# 8. Forms Design

## Principles

- Simple inputs
- Clear labels
- Minimal steps
- Validation messages must be clear

---

# 9. Status Indicators

Used in moderation:

- Pending (Yellow)
- Approved (Green)
- Rejected (Red)

---

# 10. Typography

- Clean sans-serif font
- Large headings for readability
- Comfortable line spacing

---

# 11. Colors

## Primary Theme

- Neutral dark/light theme
- Soft background
- Minimal contrast noise

---

## Emotional Safety Rule

No harsh or aggressive color usage.

---

# 12. Responsive Design

- Mobile-first approach
- Journey Page must be fully readable on mobile
- Cards must adapt to screen size

---

# 13. UX Principles

- Reduce cognitive load
- Prioritize readability
- Ensure fast navigation
- Keep interactions simple

---

# 14. Accessibility

- Proper contrast
- Readable font sizes
- Keyboard navigation support

---

# Core Principle

The UI must never distract from the story.

It exists only to make the journey readable, not impressive.
