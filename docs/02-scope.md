# Project Scope

This document defines the boundaries of the Insan platform.

The first release (MVP) focuses on delivering a reliable, respectful, and maintainable platform for documenting and preserving human journeys.

Only features defined in this scope are considered part of Version 1.

---

# In Scope

The first version includes:

## Authentication

- User registration
- User login
- JWT authentication
- Role-based authorization

---

## Human Journeys

- Create a journey
- Edit a journey
- Delete a journey
- Browse journeys
- Search journeys

---

## Journey Details

Each journey contains:

- Basic information
- Biography
- Life journey
- Dreams
- Skills
- Achievements
- Stories
- Voices
- Memories
- Legacy

---

## Memories

Support:

- Images

Future:

- Videos
- Audio

---

## Voices

Registered users can submit memories, stories, or testimonies.

Moderator approval is required before publication.

---

## Search

Search by:

- Name
- City
- Occupation

Future versions will include advanced filters.

---

## Dashboard

Administrators can:

- Manage users
- Manage journeys
- Review submissions
- Moderate content

---

## Sharing

Each journey has:

- Shareable URL
- Open Graph preview
- QR code (Future)

---

# Out of Scope

The following features are intentionally excluded from Version 1:

- AI-generated biographies
- Mobile applications
- Interactive 3D visualization
- Voice narration
- Public API
- Institutional integrations
- Family trees
- Multi-language support
- Offline mode
- Live statistics integration

These features belong to future releases.

---

# Project Goal

The objective of Version 1 is to deliver a complete and usable platform rather than an exhaustive feature set.

The system should be stable, understandable, and extendable.

---

# MVP Definition

Version 1 is considered complete when:

- Users can browse journeys.
- Users can search journeys.
- Administrators can manage content.
- Families can preserve stories.
- The platform is secure.
- The platform is deployable.

---

# Future Scope

Future releases may include:

- AI-powered search
- Timeline visualization
- Interactive maps
- Family trees
- Voice recordings
- Institution portals
- Analytics dashboard
- Mobile applications
- Public API
- OCR support

---

# Guiding Principle

Version 1 favors quality over quantity.

Every implemented feature should be complete, maintainable, and production-ready.
